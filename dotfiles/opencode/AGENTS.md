## Commit Convention

If not specified otherwise, follow [conventional commits](https://www.conventionalcommits.org/) format: `type(scope): description`

- **Common Types**: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `perf`, `ci`, `build`
- **Scope**: optional — use when change targets a specific module/package
- **Body/footer**: optional — use for breaking changes or detailed explanations
- **Subject**: imperative mood ("add feature" not "added" or "adds")
- **Length**: keep subject line under 72 characters
